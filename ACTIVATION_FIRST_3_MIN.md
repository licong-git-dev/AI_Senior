# 老人冷启动激活 · 前 3 分钟产品设计

> 落地 [PRODUCT_INSIGHTS_V2.md](PRODUCT_INSIGHTS_V2.md) **Insight #11**。
>
> 60+ 老人首次使用 AI 产品的真实流失曲线：
> ```
> 0-15s:  60% 关闭（看到一堆图标焦虑）
> 15-60s: 70% 关闭（点错怕扣钱）
> 1-3min: 60% 关闭（说话后看到大段文字）
> ```
> 设计前 3 分钟 = 决定其余所有 23 轮工作的命运。

---

## 一、3 句话激活法（核心方案）

```
🎙 第 1 句（5 秒）·【建立熟悉感】
"张妈妈，您家伢小军让我来陪您唠两句。"

✅ 关键设计:
   · 用老人本人的姓 (注册时家属填)
   · 用最亲子女的名字 (减少陌生感)
   · "唠两句" 不是 "聊天" (方言贴近)
   · 不说 "你好" / "我是 AI" (这是禁忌)
```

```
🎙 第 2 句（10 秒）·【给可消费的内容】
"我晓得您今儿个一个人在家，要不要听我给您说说今天的天气，
和晚上 7 点中央 1 套有部您上次问过的电视剧？"

✅ 关键设计:
   · 先 "推内容" 而非 "问问题"
   · 内容必须具体 (天气 + 电视剧时段)
   · 体现 "我记得您问过" (复用 MemoryEngine)
   · 给两个选项不是开放题 (老人选择恐惧)
```

```
🎙 第 3 句（15 秒）·【给操作权】
"想跟我说话，按中间绿色大按钮；想喊我闭嘴，按红色按钮。
您家慢慢来，我一直在这。"

✅ 关键设计:
   · 明确双向操作
   · 给 "闭嘴权" 降低控制焦虑 (老人怕停不下来)
   · "我一直在这" 是结尾品牌语
   · 不催促，不教程
```

**整段 30 秒。说完不再主动开口，等老人自己来。**

---

## 二、为什么"3 句话"而不是教程

### 教程模式的失败原因
- 老人对"教程"的心智 = "考试"，焦虑反应
- "下一步" 按钮被认为是"扣钱"
- 文字提示看不清（即使字号 24px）
- 跳过教程 = 老人觉得"我不会用"

### 3 句话模式的成功原理
- AI **主动开口**而不是被动等待
- 内容**具体可消费**（天气+电视剧）而非抽象问候
- 操作**只有两个键**（绿/红），降到老人能接受的复杂度
- "我一直在这"建立**陪伴而非工具**心智

## 三、必备的注册数据采集

家属在注册老人账户时**必填**：

| 字段 | 用途 | 例 |
|---|---|---|
| 老人姓 | 第一句话称呼 | 张 |
| 老人称呼 | 替换"妈妈"为更具体的（"妈"/"娘"/"婆婆"）| 妈 |
| 最亲子女称呼 | 第一句话提及 | 小军 |
| 老人方言区 | 决定 dialect_companion 路径 | wuhan / mandarin |
| 老人电视习惯 | 第二句话内容 | 中央 1 套 19:00 / 楚剧 |
| 老人健康关注点 | 个性化 health 类对话 | 高血压 / 糖尿病 |

**这 6 个字段决定 AI 在前 30 秒能不能"击中"老人**。

注册流程改造：
```
当前注册:
[ 手机号 ] [ 密码 ] [ 老人姓名 ] → 完成

应改为:
[ 手机号 ] [ 密码 ]
   ↓
[ 老人姓 ] [ 您怎么称呼老人 ]
   ↓
[ 您是老人的{son/daughter/孙辈/...} ] [ 老人怎么称呼您 ]
   ↓
[ 老人方言：选择] [ 老人喜欢看的节目 ]
   ↓
[ 老人健康关注：高血压/糖尿病/关节/睡眠/无 ]
   ↓
完成 + 自动生成"3 句话激活语音"
```

## 四、二次激活（D1-D7 持续唤回）

3 句话只解决"第一次"。第 2-7 天的留存还要：

### D1（次日早上）
**触发器**：第二天早上 9:00（用老人本地时区）
**内容**：用老人姓 + 提到昨天聊过的事
```
"张妈妈，早呀！昨天您跟我提到您家伢小军要回来过中秋，我帮您看了下日历，
还有 27 天嘞。今儿个想唠点啥？"
```

### D3（首个意外惊喜）
**触发器**：D3 任意时段（避免老人预期）
**内容**：用老人讲过的一件小事，主动跟进
```
"张妈妈，您前几天提到您家阳台桂花要开了，今儿个开了冇？我蛮想知道。"
```

### D7（建立"它真的记得我"）
**触发器**：第 7 天主动总结
**内容**：用 MemoryEngine 召回过去 7 天的高频话题
```
"张妈妈，咱们认识一周了。您这周跟我聊了好多关于小军的事，
还有您家王娭毑下棋老赢您。要不要继续讲点啥给我听？"
```

## 五、关键防止"AI 自杀"的设计

### 别犯这些错（实测会让老人秒退）
- ❌ 自报家门"我是安心宝 AI 助手"
- ❌ 长篇大论解释功能
- ❌ 选择题超过 2 个
- ❌ 任何弹窗 / 协议 / 同意按钮（家属注册时已统一搞定）
- ❌ 无内容只问"您今天怎么样？"
- ❌ 文字超过 1 行的回复（老人看不下去）
- ❌ 提到"AI / 智能 / 算法 / 数据"等技术词

### 必须做这些（实测有效）
- ✅ 主动开口，不等老人
- ✅ 第一句必含老人本人姓 + 最亲家属名
- ✅ 推具体内容而非问开放题
- ✅ 给"闭嘴"按钮（控制感）
- ✅ 用方言（哪怕只一个语气词）
- ✅ 单次回复 ≤ 30 字（已是 PersonaConfig 默认）

## 七、衡量指标

| 指标 | 当前估计 | 目标 |
|---|---|---|
| D1 留存（首次注册次日还打开）| 行业基线 25% | **>= 50%** |
| D7 留存 | 10% | **>= 25%** |
| 首次会话时长 | < 60 秒 | **>= 3 分钟** |
| 首次会话主动说话条数 | < 1 | **>= 3 条** |

D1 留存翻倍 = 整个产品 LTV 翻倍。这是 ROI 最高的一件事。
