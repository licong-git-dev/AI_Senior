## 一、日常巡检（每天 5 分钟）

### 1.1 一键健康检查脚本

将以下脚本保存为 `/opt/anxinbao/check.sh`：

```bash
#!/bin/bash
echo "====== 安心宝服务巡检 $(date '+%Y-%m-%d %H:%M') ======"

# 1. Docker 容器状态
echo ""
echo "📦 容器状态："
docker compose ps --format "table {{.Name}}\t{{.Status}}\t{{.Ports}}"

# 2. API 健康检查
echo ""
echo "🏥 API 健康："
HEALTH=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8000/health)
if [ "$HEALTH" == "200" ]; then
    echo "  ✅ API 正常 (HTTP $HEALTH)"
else
    echo "  ❌ API 异常 (HTTP $HEALTH)"
fi

# 3. 数据库连接
echo ""
echo "🗄️ 数据库："
DB_CHECK=$(docker exec anxinbao-db pg_isready -U anxinbao 2>&1)
echo "  $DB_CHECK"

# 4. Redis 连接
echo ""
echo "📮 Redis："
REDIS_CHECK=$(docker exec anxinbao-redis redis-cli ping 2>&1)
echo "  $REDIS_CHECK"

# 5. 磁盘使用
echo ""
echo "💾 磁盘使用："
df -h / | tail -1 | awk '{print "  使用率: "$5" (已用 "$3" / 总计 "$2")"}'

# 6. 内存使用
echo ""
echo "🧠 内存使用："
free -h | grep Mem | awk '{print "  使用: "$3" / 总计: "$2" (可用 "$7")"}'

# 7. 最近错误日志（最近 1 小时）
echo ""
echo "⚠️ 最近 1 小时错误日志："
docker compose logs --since 1h api 2>&1 | grep -i "error\|exception\|critical" | tail -5
if [ $? -ne 0 ]; then
    echo "  ✅ 无错误日志"
fi

echo ""
echo "====== 巡检完成 ======"
```

```bash
# 赋予执行权限
chmod +x /opt/anxinbao/check.sh

# 每天早上 9 点自动执行并发送邮件（可选）
# crontab -e
# 0 9 * * * /opt/anxinbao/check.sh | mail -s "安心宝日报" your@email.com
```

### 1.2 需要关注的指标

| 指标 | 正常值 | 告警阈值 | 处理方式 |
|------|--------|---------|---------|
| API 响应时间 | <200ms | >1000ms | 检查数据库慢查询 |
| CPU 使用率 | <60% | >85% | 考虑升级配置 |
| 内存使用率 | <70% | >85% | 检查内存泄漏 |
| 磁盘使用率 | <70% | >85% | 清理日志/备份 |
| 数据库连接数 | <80% 池大小 | >90% | 增加连接池/排查泄漏 |
| Redis 内存 | <200MB | >230MB | 检查缓存策略 |

## 二、数据备份

### 2.1 数据库自动备份

保存为 `/opt/anxinbao/backup.sh`：

```bash
#!/bin/bash
# 安心宝数据库备份脚本

BACKUP_DIR="/opt/anxinbao/backups"
DATE=$(date +%Y%m%d_%H%M%S)
KEEP_DAYS=30

mkdir -p $BACKUP_DIR

echo "[$(date)] 开始数据库备份..."

# PostgreSQL 备份
docker exec anxinbao-db pg_dump -U anxinbao anxinbao | gzip > "$BACKUP_DIR/db_$DATE.sql.gz"

if [ $? -eq 0 ]; then
    SIZE=$(ls -lh "$BACKUP_DIR/db_$DATE.sql.gz" | awk '{print $5}')
    echo "[$(date)] ✅ 备份成功: db_$DATE.sql.gz ($SIZE)"
else
    echo "[$(date)] ❌ 备份失败!"
    # 可以在这里添加告警通知
fi

# Redis 备份（RDB 快照）
docker exec anxinbao-redis redis-cli BGSAVE
sleep 5
docker cp anxinbao-redis:/data/dump.rdb "$BACKUP_DIR/redis_$DATE.rdb"

# 清理旧备份
find $BACKUP_DIR -name "*.gz" -mtime +$KEEP_DAYS -delete
find $BACKUP_DIR -name "*.rdb" -mtime +$KEEP_DAYS -delete

echo "[$(date)] 备份完成。当前备份文件："
ls -lh $BACKUP_DIR | tail -10
```

```bash
chmod +x /opt/anxinbao/backup.sh

# 设置定时任务：每天凌晨 3 点备份
# crontab -e
# 0 3 * * * /opt/anxinbao/backup.sh >> /opt/anxinbao/logs/backup.log 2>&1
```

### 2.2 备份恢复

```bash
# 恢复 PostgreSQL
gunzip < /opt/anxinbao/backups/db_20260303_030000.sql.gz | \
  docker exec -i anxinbao-db psql -U anxinbao anxinbao

# 恢复 Redis
docker stop anxinbao-redis
docker cp /opt/anxinbao/backups/redis_20260303_030000.rdb anxinbao-redis:/data/dump.rdb
docker start anxinbao-redis
```

### 3.2 简易告警脚本

保存为 `/opt/anxinbao/alert.sh`：

```bash
#!/bin/bash
# 简易告警脚本（通过企业微信/钉钉/飞书 Webhook 发送）

WEBHOOK_URL="你的Webhook地址"

# 检查 API
HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 http://localhost:8000/health)
if [ "$HTTP_CODE" != "200" ]; then
    MSG="🚨 安心宝 API 异常！HTTP 状态码: $HTTP_CODE，时间: $(date '+%H:%M')"
    curl -s -X POST "$WEBHOOK_URL" \
        -H "Content-Type: application/json" \
        -d "{\"msgtype\": \"text\", \"text\": {\"content\": \"$MSG\"}}"
fi

# 检查磁盘
DISK_USAGE=$(df / | tail -1 | awk '{print $5}' | tr -d '%')
if [ "$DISK_USAGE" -gt 85 ]; then
    MSG="⚠️ 安心宝服务器磁盘使用率 ${DISK_USAGE}%，请及时清理"
    curl -s -X POST "$WEBHOOK_URL" \
        -H "Content-Type: application/json" \
        -d "{\"msgtype\": \"text\", \"text\": {\"content\": \"$MSG\"}}"
fi

# 检查内存
MEM_USAGE=$(free | grep Mem | awk '{printf "%d", $3/$2 * 100}')
if [ "$MEM_USAGE" -gt 85 ]; then
    MSG="⚠️ 安心宝服务器内存使用率 ${MEM_USAGE}%，请检查"
    curl -s -X POST "$WEBHOOK_URL" \
        -H "Content-Type: application/json" \
        -d "{\"msgtype\": \"text\", \"text\": {\"content\": \"$MSG\"}}"
fi
```

## 五、版本更新流程

### 5.1 常规更新（不停服）

```bash
cd /opt/anxinbao

# 1. 拉取最新代码
git pull origin main

# 2. 重新构建并滚动更新
docker compose build api
docker compose up -d api

# 3. 验证
curl http://localhost:8000/health
docker compose logs --tail 20 api
```

### 5.2 数据库迁移更新

```bash
# 如果有表结构变更

# 1. 先备份
./backup.sh

# 2. 拉取代码
git pull origin main

# 3. 重建并启动
docker compose build api
docker compose up -d api

# 4. 进入容器执行迁移
docker exec -it anxinbao-api python -c "from app.models.database import init_db; init_db()"

# 5. 验证
curl http://localhost:8000/health
```

### 5.3 回滚操作

```bash
# 回滚到上一个版本
git log --oneline -5           # 查看最近提交
git checkout <上一个commit>    # 切换版本
docker compose build api
docker compose up -d api

# 如果需要恢复数据库
gunzip < /opt/anxinbao/backups/db_最近的备份.sql.gz | \
  docker exec -i anxinbao-db psql -U anxinbao anxinbao
```

## 七、故障处理速查表

| 故障现象 | 可能原因 | 处理步骤 |
|---------|---------|---------|
| API 无响应 | 容器挂了 | `docker compose restart api` |
| API 返回 500 | 代码异常 | `docker compose logs api` 查看错误 |
| API 返回 429 | 触发限流 | 检查是否被攻击；调整限流阈值 |
| 数据库连接失败 | PG 挂了/满了 | `docker compose restart db`；检查连接数 |
| Redis 连接失败 | Redis 内存满 | `docker exec anxinbao-redis redis-cli INFO memory` |
| 磁盘满 | 日志/备份堆积 | 清理日志；删除旧备份 |
| SSL 证书过期 | 未自动续期 | `certbot renew` 然后重启 nginx |
| AI 对话无回复 | API Key 过期/欠费 | 检查灵积控制台余额 |
| 方言识别失败 | 讯飞额度用完 | 检查讯飞控制台额度 |

```bash
sudo apt update && sudo apt upgrade -y

# 每月一次：Docker 镜像更新
```

### 8.2 安全检查清单

- [ ] SSH 已禁用密码登录，仅允许密钥登录
- [ ] 防火墙只开放必要端口（22/80/443）
- [ ] 数据库端口不对外暴露
- [ ] `.env` 文件权限设为 600
- [ ] JWT 密钥足够长（>=32 字符）
- [ ] DEBUG=false
- [ ] 定期检查 `docker compose logs` 是否有异常请求
