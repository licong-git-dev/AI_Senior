# 商业模式 · 订阅之外 · GMV + 佣金 + 数据三轮驱动

> 落地 [PRODUCT_INSIGHTS_V2.md](PRODUCT_INSIGHTS_V2.md) **Insight #13**。
>
> ¥588/年订阅天花板撑不起 BD/技术成本。但老人月可支配消费 ¥150-300（药品/保健品/营养品/服务）正在花，**只是没流过我们**。

---

## 一、为什么订阅模型必败

### 单户订阅的天花板
- ¥29/月 × 12 = ¥348/年（基础）
- ¥69/月 × 12 = ¥828/年（高级）
- ¥588/年（年卡）
- 实际转化后**平均 ARPU ≈ ¥350/年**（含免费/试用稀释）

### 单户成本下限
- 通义千问 + 讯飞调用 ≈ ¥30-80/月/老人
- BD 获客（CAC）≈ ¥30-50/老人（线上）+ ¥10-20（社区）
- 客服 ≈ ¥10/月/老人（老人需要的人工干预多）
- 推送/短信/通话 ≈ ¥5-15/月/老人

**月成本 ≈ ¥55-115，年成本 ≈ ¥660-1380**。

**订阅 ARPU < 单户年成本**。订阅模式独立看不可持续。

### 行业类比
- ElliQ：硬件 ¥7000 + 订阅 ¥240/月 → 高客单
- 小度音箱：硬件低 + 广告变现
- character.ai：免费 + 高级 ¥75/月（服务的不是老人，是有手机消费力的年轻人）
- **没有任何独立靠"老人订阅"赚钱的产品**

## 二、三层变现叠加：GMV + 佣金 + 数据

### A 层 · 健康好物商城（最先落地）

#### 核心
**AI 在对话中识别老人需求 → 推荐商品 → 子女付款**

#### 老人侧体验（不打扰）
```
[场景：老人对 AI 说]
"最近腿没劲，下楼都费劲"

[AI 不直接推商品，而是关切]
"妈妈，您腿没劲多久了？要不要我帮您查查最近的血钙？"

[查后]
"您上个月的血钙偏低，可能是缺钙。我跟小军说一声，
让他帮您安排一下，好不好？"

[老人]
"好"

[AI]
"好咧。我已经告诉小军了。"
```

#### 子女侧体验（被通知）
```
推送：「妈妈最近腿没劲，AI 检查发现可能缺钙
       【推荐】钙片 (适合 60+ 老人) ¥89
       【一键下单】 → 配送到家
       【先看医生】 → 跳预约页」
```

#### 关键不是商城，是 **"AI 翻译模糊需求 → 具体 SKU"** 的能力
- 老人说"睡不好" → 不卖安眠药（违法），AI 推荐"褪黑素"或"助眠枕"
- 老人说"皮肤痒" → AI 推荐"老人护肤霜"
- 老人说"耳朵嗡嗡响" → AI 推荐"先去查听力"，而不是直接卖助听器

#### 商业逻辑
- 抽佣 5-15%，单笔毛利 ¥5-50
- 月 GMV/老人 ≈ ¥50-150 → 月分成 ¥2.5-22.5
- 1000 老人 → 月增量收入 ¥2500-22500
- **不替代订阅，叠加在订阅上**

#### 实施路径
- **不要立刻自建商城**
- 第 1 阶段：识别意图 + 标记（已可用 LLM 完成）
- 第 2 阶段：接淘宝/京东老年板块联盟链接（抽佣低但快）
- 第 3 阶段：累积 6 个月数据后，自营高频 SKU（毛利高）

---

### B 层 · 社区服务订单分成

#### 核心
对接江岸区社区/医院的真实服务（理发/送菜/血压车/保洁/代购药），**老人发起 → 安心宝下单 → 服务方分成**。

#### 已有基础
- [`integration_service.book_appointment`](anxinbao-server/app/services/integration_service.py)
- [`book_community_service`](anxinbao-server/app/services/companion_tools.py) 工具（在 `IntegrationNotImplemented` 守卫后）
- 当前生产环境会抛异常 → 需要接通真实社区方

#### 商业逻辑
- 单笔分成 ¥5-30
- 月频次 1-3 单/老人 → 月分成 ¥5-90
- 1000 老人 → 月增量 ¥5000-90000

#### B2B2C 优势
- 社区方愿意分（他们获客难）
- 老人接受度高（已经在用社区服务）
- **政府背书**：江岸民政局可推荐

#### 实施路径
- 与江岸 1-2 个社区先签 PoC
- 上线 5-10 个最高频服务（理发/送菜/代购药为主）
- 跑 3 个月看复购率

### C 层 · 数据资产服务化（最高毛利，最难做）

#### 核心
**经多重授权后**，把脱敏聚合的老年人健康行为数据卖给保险公司/医药企业/区域医院。

#### 数据池能产出什么
```
个人维度（永远不出池）：
  · 老人 ID 1234567 是 72 岁高血压

群体维度（合规可外售）：
  · 江岸区 65-75 岁高血压老人，
    用药依从性 = 78%（行业平均 60%）
  · 安心宝陪伴用户 vs 非用户的
    急诊就医次数差异 = -23%
```

#### 行业付费意愿
- 老年险定价模型：单个精准画像 **¥50-500/年**
- 慢病用药依从性研究：**¥10-50 万/项目**
- 区域老年健康画像：**¥20-100 万/年/省级保险公司**

#### 红线（不可逾越）
1. 老人本人 + 主 payer 双重授权
2. 仅出群体维度，不出个人维度
3. 数据可携带（老人/家属一键导出全部原始数据，可一键销毁）
4. 变现分成：每条数据被使用，老人账户加陪伴值或现金抵扣

#### 实施路径（**长线**，不在 r25 范围）
- r25-r28：积累数据 + 建脱敏管道
- r29+：法务过 RFC + 试点客户
- 6 个月内不计划接客户

## 三、综合 ARPU 模型

### 当前订阅模型
```
ARPU = ¥350/年
成本 = ¥800/年
亏损 = ¥450/老人/年（≈ -¥38/月）
```

### r25 → r28 GMV+佣金叠加后
```
ARPU = 订阅 ¥350 + GMV 抽成 ¥150 + 服务分成 ¥200
     = ¥700/年
成本 = ¥800/年（不变）
仍亏损 = ¥100/老人/年
```

### r28 → r30 数据资产成熟后
```
ARPU = ¥700 + 数据分成 ¥100
     = ¥800/年（基本盈亏平衡）
+ 政府/B 端项目 ¥50万-200万/年（覆盖固定成本）
```

### 真正的拐点
- 1000 老人内：靠 B 端项目（民政局/保险）补贴
- 1000-10000 老人：GMV+佣金支撑可持续
- 10000+ 老人：数据资产收益开始显现

## 四、不要做的事

### ❌ 不要做"老人订阅升级"
- 老人对"按月扣费"极度反感
- ¥588 已经是天花板
- 推 ¥888/¥1500 套餐 = 流失加速

### ❌ 不要做"广告位贴片"
- 老人看到广告 = 不信任
- 必须是 AI 用方言推荐而非视觉广告

### ❌ 不要做"打赏 / 礼物"
- 这是年轻人产品的玩法，老人不接受

### ❌ 不要"什么都卖"
- 商城品类必须严格筛选：
  - 老人需要、子女理解、客单 ¥50-300
  - 高频复购 > 一次性
  - 无医疗法律风险

## 五、第一步该做什么（r25 之后 1-2 月）

### Step 1：意图识别（无成本，立即做）
- 在 Hermes.chat 末尾增加"商业意图标记"
- 老人对话中提到"想买/最近缺/有点贵"等关键词 → 标记
- 不下单，仅记录 → 6 个月后看分布

### Step 2：与子女打通（4 周）
- 子女端 ChildDashboard 增加"妈妈最近想要的东西"卡片
- 来源就是 Step 1 标记
- 加"看看"按钮 → 跳到联盟链接

### Step 3：与社区打通（2 个月）
- 江岸 1-2 个社区签 PoC
- 上线 3-5 个高频服务

### Step 4：数据合规先行（4 个月）
- 法务过数据合约模板
- 用户协议加"数据共享授权"分级开关

## 七、与 24 轮已建能力的连接

GMV+佣金模型直接复用：

- **HealthAgent / ScheduleAgent**：判断时机

**24 轮的基础设施都被这个商业模型消费**。一行代码不用改架构。
