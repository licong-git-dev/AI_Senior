## 一、健康好物联盟（GMV 漏斗的"出口层"）

### 为什么联盟而不是自营商城
- 自营 = 仓储 / 客服 / 售后人力 ≥ ¥30万/年
- 联盟 = 0 库存，分成 5-15%
- 适配老人 SKU 需要 6-12 个月用户调研，先借联盟方供应链

### 目标合作品牌（按优先级）

| 类别 | 推荐品牌 | 抽佣范围 | 备注 |
|---|---|---|---|
| 营养品 | 汤臣倍健 / 善存 / Swisse | 8-15% | 钙片/维 D/复合维生素 |
| 镇痛/关节 | 腾药 / 云南白药 | 10-20% | 护膝/膏药 |
| 助眠 | 网易严选 / MUJI | 10-15% | 助眠枕/蒸汽眼罩 |
| 防滑/扶手 | 京东居家 / 拼多多老年专区 | 5-10% | 浴室扶手/防滑垫 |
| 大字版图书 | 当当 / 京东图书 | 5-8% | 老花镜/读书机 |
| 智能血压计 | 欧姆龙 / 鱼跃 | 8-12% | 复购率高 |

### 接入流程（4 步上线一个 SKU）

```
Step 1 · 选品（产品 + BD 协作 · 1 周）
  · 用 GET /api/companion/intents/stats?elder_user_id=* 拉本周高频意图
  · 优先 confidence >= 0.85 + 家属 review > 30% 的关键词
  · 砍掉伦理风险高的（如未明确效果的"保健品""神药"）

Step 2 · 接洽供应商（BD · 1-2 周）
  · 拿到联盟 API + 抽佣率
  · 谈"AI 推荐场景独家分成"溢价（普通联盟 5% → 我们 8-12%）
  · 必须含：
    - 7 天无理由退换货（老人误买的安全网）
    - 子女付款不需老人确认（避免老人在 AI 引导下乱花钱）
    - 失败赔付标准（订单丢失 → 全额退）

Step 3 · 上架（工程 · 1-2 天）
  · 在 commercial_intent_service.REDEMPTION_CATALOG 添加 SKU
  · 字段：sku_id / title / image_url / price / partner_url / commission_pct
  · CommercialIntent 表 status 加 ordered 状态
  · 子女端 IntentCard 加"立即下单"按钮 → 跳联盟方
  · 跟踪订单回调（partner webhook → 我们 record commission）

Step 4 · 复盘（运营 · 每月）
  · 转化率：detected → reviewed → ordered 漏斗
  · 客单价 / 复购率 / 退款率
  · 砍掉 30 天 0 转化的 SKU，引入新 SKU
```

### 关键合作条款模板（BD 用）

```
甲方：安心宝（AI 智能养老平台）
乙方：[品牌方]

合作内容：
1. 乙方授权甲方在 AI 对话中根据老人提及的健康需求推荐乙方 SKU
2. 推荐场景：仅向有真实需求迹象的子女展示（基于 confidence ≥ 0.85）
3. 不向老人本人展示商品（老人保护条款 1）

付费模式：
- CPS（按成交分成）：销售额 × {agreed}% (8-15%)
- 不向甲方收取入场费 / 推广费
- 月结，乙方在每月 5 日前支付上月分成

老人保护条款：
1. 任何 SKU 推荐**不直接向老人展示**，仅子女端可见
2. 任何 SKU 必须支持 7 天无理由退换货
3. 因 AI 误推荐导致老人或家属经济损失，乙方按销售额 2 倍赔付
4. 乙方不得向甲方索要老人个人健康数据
5. 乙方收到退货必须在 3 个工作日内退款

数据合规：
- 甲方仅向乙方传递 SKU 关键词与转化数据（不传老人姓名/手机号/详细聊天）
- 乙方收集的订单信息归乙方所有
- 双方各自承担 GDPR / 个人信息保护法 责任

终止条款：
- 任一方提前 30 天书面通知可终止
- 历史订单分成义务持续至全部售后期满
```

## 二、江岸区民政局 PoC 战略

### 为什么 B2G 是真正的 0 → 1
- **背书天花板**：政府名头老人最信
- **CAC = ¥0**：政府主动推
- **付费方明确**：政府智慧养老券预算
- **政策红利**：江岸区 60+ 老人 ≥ 12 万

### PoC 三步走

```
Step 1 · 接洽（BD · 1-2 周）
  · 联系人：江岸区民政局 智慧养老办
  · 提案：免费提供 100 个账户给街道试点
  · 不谈钱，先谈"为政府解决什么问题"
  · 准备材料：
    - ROADSHOW_SUMMARY.md（产品介绍）
    - DEMO_SCRIPT.md（30 分钟现场演示脚本）
    - 数据安全合规承诺函（见 docs/legal/）

Step 2 · 试点（运营 + 工程 · 2-3 月）
  · 选 1-2 个街道（如百步亭、二七）
  · 现场招募 100 老人 + 工作人员代填 5 onboarding 字段
  · 工程支持：custom 分支 / 部署独立环境
  · 每周给民政局发数据简报：
    - 老人活跃天数 / 主动对话次数
    - SOS 触发次数 / 响应时间
    - 子女注册率 / NPS

Step 3 · 转化（BD · 第 4 月起）
  · 数据漂亮 → 谈批量采购 / 智慧养老券补贴
  · 单价 ¥200-500/老人/年（B2G 高于 C 端）
  · 政府采购 = 公开招标 → 必须有同类型成功案例 → PoC 数据是关键
```

### 试点合同模板要点

| 字段 | 标准 |
|---|---|
| 试点周期 | 3-6 个月 |
| 试点用户数 | 100-500 |
| 数据归属 | 老人所有 + 政府聚合查询权（不含个人） |
| 退出机制 | 老人/家属随时可一键删除全部数据 |
| 合规 | 通过等保 2 级 + 网信办备案 |
| 保险 | 试点期间老人因 AI 错误指引导致损失，平台承担最高 ¥10万 |
| KPI | D30 留存 ≥ 50% / NPS ≥ 30 / SOS 响应 < 5min |

## 六、附：联盟方拒绝率高的话术

**问**：为什么我们要给你们溢价分成？
**答**：因为我们的推荐**只发生在子女有真实付款意愿的场景**。普通联盟流量 1% 转化，我们 8-15%。算下来溢价分成更高 ROI。

**问**：你们有多少用户？
**答**：试点期 100-500（江岸区民政局背书）。但每户客单价 ¥150-300/月，年化 ¥1800-3600。

**问**：万一老人退货太多怎么办？
**答**：每条 SKU 必须支持 7 天无理由 + 老人误买全额赔。我们的 IntentCard 已经做了"看到了 / 不用买"的过滤层，子女主动决策买 vs 关闭。
