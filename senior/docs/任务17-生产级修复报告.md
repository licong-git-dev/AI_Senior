### 2. 移除热力图随机数填充
**问题**: 数据不足时使用Math.random()填充  
**修复**: 
- 完全移除随机数生成逻辑
- 仅使用真实的behavior_patterns和sensor_data
- 数据不足时显示明确提示

**代码变更**:
```typescript
// 修复前（随机填充）
if (data.length < 50) {
  for (let d = 0; d < 7; d++) {
    for (let h = 0; h < 24; h++) {
      if (!data.find(item => item[0] === h && item[1] === d)) {
        data.push([h, d, Math.round(Math.random() * 100)]) // ❌ 假数据
      }
    }
  }
}

// 修复后（真实数据 + 提示）
// 只使用真实数据，不填充
const hasData = data.length > 0
const dataCompletion = (data.length / (7 * 24)) * 100

return {
  title: { 
    text: hasData ? '24小时活动热力图' : '24小时活动热力图（数据不足）',
    subtext: hasData ? `数据完整度: ${dataCompletion.toFixed(1)}%` : '暂无足够数据展示'
  },
  // ...
  series: [{ 
    data: hasData ? data : [], // ✅ 仅显示真实数据
  }]
}
```

**数据来源**:
1. behavior_patterns.activity_trajectory.activeTime
2. sensor_data (data_type='activity' 或 'steps')
3. 步数数据归一化到0-100范围

### 3. 添加数据完整性验证
**问题**: 用户无法知道数据是否完整  
**修复**: 
- 新增数据完整性状态面板
- 实时显示4个数据源状态

**新增UI组件**:
```tsx
<div className="mb-6 p-4 bg-blue-50 border-l-4 border-blue-500 rounded-lg">
  <h4 className="text-sm font-semibold text-blue-900 mb-2">
    数据完整性状态
  </h4>
  <div className="grid grid-cols-2 md:grid-cols-4 gap-3 text-xs">
    {/* 4个状态指示器 */}
    <div className={healthData.length > 0 ? 'text-green-700' : 'text-gray-500'}>
      <span className={`w-2 h-2 rounded-full ${healthData.length > 0 ? 'bg-green-500' : 'bg-gray-400'}`}></span>
      健康数据: {healthData.length > 0 ? `${healthData.length}条` : '暂无数据'}
    </div>
    {/* ...其他3个指示器 */}
  </div>
  {/* 数据缺失警告 */}
  {(healthData.length === 0 || physiologicalData.length === 0) && (
    <div className="mt-3 text-xs text-orange-700 bg-orange-50 p-2 rounded">
      ⚠️ 部分数据缺失，图表可能显示不完整。请确保已有足够的历史数据。
    </div>
  )}
</div>
```

**状态指示器**:
- 🟢 绿点 + 绿色文字 = 有数据
- ⚪ 灰点 + 灰色文字 = 无数据
- 🟠 橙色背景警告 = 数据不足
