## 新增功能概览

### 1. 老人端交互功能 🎯

#### 实时健康监测
- **心率监测**: 实时显示当前心率（bpm），每3秒自动更新
- **血压监测**: 实时显示收缩压/舒张压（mmHg），动态变化
- **步数统计**: 实时累计步数，模拟运动数据
- **体温监测**: 实时显示体温（℃），精确到0.1度

#### 一键交互操作
1. **模拟跌倒检测**
   - 点击触发跌倒事件模拟
   - 自动记录到操作日志
   - 2秒后自动跳转到护理管理端查看紧急呼叫
   - 模拟真实的跌倒检测和报警流程

2. **紧急呼叫**
   - 模拟老人主动求助场景
   - 即时发送紧急呼叫信号
   - 通知家属和护理人员
   - 记录呼叫时间和原因

3. **健康数据上传**
   - 模拟设备数据同步
   - 上传当前实时监测的心率和血压
   - 显示上传成功反馈
   - 记录上传时间戳

#### 操作记录日志
- 实时记录所有操作事件
- 按时间倒序显示
- 不同事件类型使用不同颜色标识
  - 跌倒检测：红色边框
  - 紧急呼叫：橙色边框
  - 数据上传：绿色边框

### 2. 实时数据更新机制 ⚡

#### 技术实现
- 使用 React `useEffect` Hook 实现定时器
- 每3秒自动更新一次健康数据
- 模拟真实IoT设备的数据采集频率

#### 数据变化规律
- **心率**: 在当前值基础上 ±3 bpm 随机波动
- **血压**: 
  - 收缩压: 110-140 mmHg 范围内随机变化
  - 舒张压: 70-90 mmHg 范围内随机变化
- **步数**: 每次更新增加 0-20 步
- **体温**: 在当前值基础上 ±0.1℃ 随机波动

### 3. 原有功能保持 ✅

- **系统概览**: 50个老人档案、统计数据、社区分布
- **家属监控端**: ECharts血压心率趋势图、健康记录表
- **护理管理端**: 紧急呼叫列表、高风险老人管理
- **自动演示模式**: 4个场景自动轮播（每5秒切换）

## 使用场景演示

### 场景1: 日常健康监测
1. 进入演示页面，点击"老人端"Tab
2. 观察实时健康数据卡片
3. 等待数秒，查看数据自动更新
4. 点击"上传健康数据"按钮同步数据

### 场景2: 跌倒检测响应
1. 在老人端点击"模拟跌倒检测"按钮
2. 系统显示跌倒警报消息
3. 自动跳转到护理管理端
4. 查看紧急呼叫记录中的跌倒事件

### 场景3: 主动紧急呼叫
1. 在老人端点击"紧急呼叫"按钮
2. 查看操作记录中的呼叫确认
3. 切换到护理管理端查看呼叫处理状态

### 场景4: 家属远程监控
1. 切换到"家属监控端"Tab
2. 选择需要查看的老人
3. 查看血压和心率趋势图（最近7天）
4. 浏览最新健康记录表格

## 技术架构

### 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite 6
- **UI样式**: TailwindCSS
- **图表库**: ECharts + echarts-for-react
- **图标库**: Lucide React
- **路由**: React Router DOM

### 状态管理
- React Hooks (useState, useEffect)
- 实时数据状态: `realtimeData`
- 事件记录状态: `simulatedEvents`
- Tab切换状态: `activeTab`

### 核心组件
```typescript
// 实时数据结构
interface RealtimeData {
  heartRate: number;        // 心率
  bloodPressure: {
    systolic: number;       // 收缩压
    diastolic: number;      // 舒张压
  };
  steps: number;            // 步数
  temperature: number;      // 体温
}

// 事件记录结构
interface SimulatedEvent {
  id: string;               // 事件ID
  type: 'fall' | 'emergency' | 'data';  // 事件类型
  message: string;          // 事件消息
  time: Date;               // 事件时间
}
```
